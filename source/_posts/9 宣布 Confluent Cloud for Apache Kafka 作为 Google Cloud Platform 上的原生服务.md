# 9 宣布 Confluent Cloud for Apache Kafka 作为 Google Cloud Platform 上的原生服务

我们很高兴地宣布，我们正在与 Google Cloud 合作，将 Confluent Cloud——我们提供的完全托管版 Apache Kafka®——作为原生服务推出在 Google Cloud Platform (GCP) 上。这意味着您将能够使用 Confluent Cloud 托管的 Apache Kafka 服务，并且可以借助熟悉的 Google 工具和流程来操作，包括集成到 Google Cloud 控制台和 GCP 市场，实现无缝注册体验，并由 Google Cloud 提供集成计费和一线支持。

我想分享一些关于为什么我认为这是一个重要举措的想法。

开源和云技术可以说是过去五年中对数据基础设施软件影响最大的两大力量。开源为企业提供了标准化且创新的平台，避免了供应商锁定的风险。云原生数据系统则赋予企业灵活性，让它们只需为实际使用的资源付费，按需扩展，并避免了本地数据系统高昂的管理成本。

然而，在云端使用托管的开源产品的体验往往并不理想。大多数开源软件的设计并未具备云原生数据系统的关键特性。所谓的“托管”云服务，通常只是将这些软件部署在服务器上进行托管，缺乏弹性扩展、灵活定价、强大的服务级别协议 (SLA) 和丰富的管理生态系统。

这使得采用云服务的企业面临两难选择：它们要么选择具有云原生特性的专有云服务，要么选择这些半托管的开源产品，后者虽然具备丰富的开源生态系统和良好的可移植性，但却无法提供真正的云服务的诸多优势。

其实，企业不应该面临这种选择。一个更好的模式正在逐渐成型：将开源项目转型为真正弹性的云数据系统是一项艰巨的任务，而能够实现这一点的最佳团队正是那些开发这些开源软件的团队。这些团队可以将企业希望的云原生能力带入它们偏好的开源平台中。

许多早期的开源公司都来自周期性发布软件并支持本地部署的世界，因此在构建真正的云服务时面临困难。但现在，新的公司正在崛起，这些公司将云作为其原生的运作环境。

Confluent 就是这样一家公司。Confluent 的创始人不仅编写了 Apache Kafka 的原始代码，还在大规模环境下将其作为服务运行。我们在构建 Confluent Cloud 时，选择了挑战自我，致力于将其打造成真正的云原生数据系统。

事实上，Confluent 已经将自身的开发流程转变为“云优先”，即代码首先发布到我们的云平台，然后才发布到我们的本地软件发行版中。这种模式让我们的云端客户能够以 SaaS 解决方案的速度获得新功能，同时也让我们的本地客户能够享受在大规模云平台上经过考验的功能。此外，这些客户还可以与全球最大 Kafka 部署之一的运营商合作。

今天，我们与 Google Cloud 的联合公告标志着这一愿景的实现迈出了重要一步。通过这次合作，客户将能够使用由专家构建的云原生开源数据系统，并且作为 GCP 的原生服务完全集成。Confluent Cloud 将与 Google Cloud 控制台集成，并在 GCP 市场上提供，客户将享受到与 Google 产品一致的无缝注册体验，并由 Google Cloud 提供计费和一线支持。

我们并不是唯一与 Google Cloud 合作的开源公司。Elastic、MongoDB、DataStax、InfluxData 和 Neo4j 也在与 Google 合作，将它们的服务作为 GCP 的原生服务推出。这是一种开源公司与云平台之间协作的优秀模式，我赞赏 Google Cloud 采取这一战略。我认为，这种开放性充分证明了他们正在构建的生态系统的活力，以及未来将会提供的丰富选择。

这一完整的集成将在未来几个季度内逐步完成。但在此期间，任何有兴趣体验该服务的人现在就可以[注册 GCP 上的 Confluent Cloud](https://www.confluent.io/get-started/)。
